# Track A Treasury Toolchain Scaffold Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Establish a working WSL-based Acton project scaffold for the Track A Treasury-first prototype.

**Architecture:** Acton does not support native Windows, so this plan first validates WSL and installs Acton there. It then scaffolds an Acton empty project into the existing repository root, preserves public-repo ignore rules, and verifies the generated baseline with `acton build`, `acton test`, `acton check`, and `acton fmt --check` before any Treasury contract work begins.

**Tech Stack:** WSL Ubuntu 22+, Acton CLI, Tolk, Git, PowerShell as host shell, Bash inside WSL.

---

## Scope Check

This plan intentionally does not implement `Treasury.tolk` yet.

Reason: the current host environment is native Windows and `acton` is not installed. Acton documentation states native Windows is unsupported and Windows users should use WSL with Ubuntu 22 or newer.

This plan produces a working, testable Acton baseline. The next plan will implement the actual Track A `Treasury.tolk` contract once the generated scaffold, paths, and template conventions exist locally.
